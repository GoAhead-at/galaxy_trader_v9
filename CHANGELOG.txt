GalaxyTrader - Changelog
========================================

Version 0.15.21 (2026-07-24)

GalaxyTrader MK4-2 (architecture / maintainability):
CHANGE: **Shared offer refresh lib** - `lib.gt.mk42.offer.refresh` is the single BS-aware buy/sell offer re-resolve (vanilla `excludeempty=false` + desiredamount/cargo deficit). Wired into fresh-offer filter, pre-validate, and final validation. Yields 1ms before return (fresh-filter `do_all` would otherwise wake-storm); never call during SELL/BUY queueing (`order.trade.galaxytradermk42.xml`).
CHANGE: **Shared free-supplier / funds libs** - `lib.gt.mk42.free.supplier` (PlayerFreeSupplier / FreeDelivery) and `lib.gt.mk42.customer.funds` (player-customer money gate). Order clamp/profit/exec consume them; cache search keeps the funds gate **inline** (mid-scan `run_script` of instant-return libs wake-storms the AIDirector). No wait in these libs (`order.trade.galaxytradermk42.xml`, `lib.gt.mk42.cache.search.xml`).
CHANGE: **Shared virtualmoney exec lib** - `lib.gt.mk42.exec.virtualmoney` creates/orders/removes temporary free offers for player-supplier buy and sell legs. No wait (must not yield while the multitrade SELL-then-BUY queue is still being built) (`order.trade.galaxytradermk42.xml`).
CHANGE: **Shared validate.reach / validate.amounts** - `lib.gt.mk42.validate.reach` unifies dock + path (escape/BL memo) + home-range for sell-only and normal candidates. `lib.gt.mk42.validate.amounts` owns clamp, free-route buyer_money / dead-SellOffer bypass, amount sync, min-cargo util, and profit gate. No wait in either lib; order keeps reject counters, cache flags, final dock recheck, and SELL/BUY `create_trade_order` sequencing (`order.trade.galaxytradermk42.xml`).
NOTE: Diagnose (MD) still mirrors free/funds rules inline - MD cannot `run_script` AI libs; keep semantics aligned when changing the libs.

GalaxyTrader MK4-2 (player-supplier free delivery):
FEATURE: **Player-owned supplier → player BS/stations is free** - when the supply source is player-owned, buy and sell legs execute via temporary `create_trade_offer` with `virtualmoney="true"` (vanilla mining/barter pattern). No wallet debit on pickup; no customer credits required on delivery. NPC supplier → player customer stays paid (`order.trade.galaxytradermk42.xml`).
CHANGE: **Funds / FreeDelivery gate** - free when `buyfree` OR player supplier; NPC + empty/low BS or station account still fails `customer_no_funds`. Tradematch stamps `$PlayerFreeSupplier` on supply pairs with a player buy station; cache search and diagnose mirror the same rule (`lib.gt.mk42.tradematch.xml`, `lib.gt.mk42.cache.search.xml`, `gt_context_diagnose.xml`).
CHANGE: **Logbook / UI-due for free legs** - `MoneyRaw=0`, UI-due 0, free-leg copy; `$PlayerFreeSupplier` read before UI-due in trade logging (`gt_trade_logging.xml`, `order.trade.galaxytradermk42.xml`).

MK3 / MK4-2 Trading (paired trades / clearance):
FIX: **False `[CL]` after multitrade buy** - buy-completion SellGate no longer cancels a queued sell because `find_buy_offer` reports `not .available` (player customers / BS demand often look unavailable while TradePerform still holds the reservation). Cancel only when the sell partner is missing or non-operational. Station resolve prefers this ware's pending sell `tradedeal` owner/buyer; `GT_ActiveTrade` is same-ware fallback only (single primary slot was wrong for secondary legs). Soft-probes stay debug-only (`gt_buy_tracking.xml`).

GalaxyTrader MK3 (Market Fabric / tradesearch):
FEATURE: **Neutral fleet cache write** - `cache_refresh` builds the home-range opportunity set without travel blacklist on pathfinding (threat strip skipped on fill). Per-ship BL is enforced at cache read from the reader's current sector via `lib.gt.cache.score_for_ship` (`lib.gt.pathfinding.xml`, `order.trade.galaxytrader.xml`).
CHANGE: **Cache read BL gate** - `score_for_ship` drops non-positive `$ShipScore` rows (neutral write may include BL-blocked pockets); returns `$Found=false` when none remain. MK3 cache hit requires positive ship score (`lib.gt.cache.score_for_ship.xml`, `lib.gt.cache.search.xml`).
CHANGE: **Cargo clearance (CL) selects from ship position** - sell_only path spaces and ranking use the ship's current sector (BL-aware). Buyer must still be within MaxSell jumps of order home. Unreachable/blacklisted near-home buyers no longer outrank reachable dumps from where the ship actually is (`lib.gt.search.sell_only.xml`, `order.trade.galaxytrader.xml`, `order.trade.galaxytradermk42.xml`).
FIX: **Cache-exhaust→live no longer skipped on first reject** - reject savegame guard tested `$tradesRejectedPathSafe?` before that copy existed, so every normal reject looked like a mid-resume and jumped to `no_trade_retry` without ForceLive. Guard now tests `$tradesRejectedPath?` (MK3/MK4/MK4-2) (`order.trade.galaxytrader.xml`, `order.trade.galaxytradermk4.xml`, `order.trade.galaxytradermk42.xml`).

GalaxyTrader MK4-2:
FIX: **Build-storage deliveries no longer flip to the parent station** - BS rows stamp `DeliveryStationId` as the base station idcode (same as the station), so sector buy-offer rescans could bind the pier instead of build storage. Refresh now follows vanilla `trade.station`: recover `SellStationBase.buildstorage` and `find_buy_offer buyer=BS` with bare `offeramount` / `desiredamount` (no idcode offer scan). Cache/diagnose resolve the parent station then use `.buildstorage` (`order.trade.galaxytradermk42.xml`, `lib.gt.mk42.cache.search.xml`, `lib.gt.mk42.live.search.xml`, `gt_context_diagnose.xml`).
CHANGE: **Neutral cache write + read filter** - persists full neutral tradematch set (`$mk42MatchTradeListForCache`) for shared cache write instead of writer depth caps; cache hit re-filters BL reachability via `score_for_ship` before `mk42.score.trades` (`order.trade.galaxytradermk42.xml`, `lib.gt.pathfinding.xml`).
FIX: **Funded build storage no longer leaves MK4-2 stuck until priority poke** - live Done releases the MK42 semaphore as `mk42live` (with Done home sector + stale-slot self-heal on request). Empty-BS money is no longer stamped into the shared pool as a permanent hard reject; cache miss forces live. Settings Clear Trade Cache also clears `GT_TradeCacheMK42` (`order.trade.galaxytradermk42.xml`, `gt_tradesearch_scheduler.xml`, `lib.gt.mk42.cache.write.xml`, `lib.gt.mk42.cache.search.xml`, `lib.gt.mk42.tradematch.xml`, `glx_lib_settings_menu.xml`).
CHANGE: **Empty build storage ranks ahead of fuller yards** - supply score is urgency-first (not linear x open buy amount), with a live destination emptiness boost from BS free/capacity so empty yards (e.g. Grand Exchange) outrank nearer half-full HQ for all wares including Claytronics (`lib.gt.mk42.tradematch.xml`, `lib.gt.mk42.score.trades.xml`).
FIX: **Dead BS OfferInvalid no longer loops the same cache rows** - OfferInvalid is a hard cache exhaust (ForceLive); only DemandReserved-only may soft-retry cache. `GT_Cache_Flag_Ship` matches BuyOffer+SellStation when SellOffer is null/dead. Sell clamp skips null SellOffer (`!= null` after `set null`) instead of treating it as present (`order.trade.galaxytradermk42.xml`, `order.trade.galaxytradermk4.xml`, `gt_trading_execution.xml`).
CHANGE: **searchDepth is per delivery class** - when a class prio is > 0, keep up to `searchDepth` trades for that class (e.g. BS=10 ST=6 → up to 30 BS + 30 ST), emit higher-prio class first for validation. Flat first-N remains single-class / non-MK4-2 only (`order.trade.galaxytradermk42.xml`).

Threat Avoidance / Sector Blacklist:

General (performance):

GalaxyTrader MK1:
FIX: **Multitrade sell legs no longer execute in reverse planned order** - sector compaction sorts sell destinations nearest-home-first, but SELL/LOAD `create_trade_order` with `immediate="true"` (LIFO) ran the farthest sell first. Contraband could reach an illegal sector while other planned wares were still aboard (e.g. Spacefuel to Hatikvah's Choice III before unloading at Hatikvah's Choice I). Queue create now uses reverse planned index (MK4-2 parity); logbook MultitradeLegs stay in planned forward order (`order.trade.galaxytradermk1.xml`).
FIX: **Buy Cargo Fill Target is a minimum, not a cargo cap** - supply runs no longer clamp load size to `buytarget%` of hold (e.g. 80% -> 3200 of 4000). Amount fills up to full hold when demand and supply allow; candidates below the floor are rejected unless remaining station need is smaller than the floor (`order.trade.galaxytradermk1.xml`, `{77000,30116}`).

GalaxyTrader MK2:
FIX: **Idle Parking release 0 no longer deadlocks docked traders** - with global Idle Parking Release Minutes at 0, `$GT_IdleParkingHold` has no HoldUntil and `orders.base` blocks undock until Hold clears. MK2 cleared Hold only after a successful trade cycle that needed undock first. Hold now releases when the trade queue is committed, before `create_trade_order` (`order.trade.galaxytradermk2.xml`).

General (performance):
FIX: **Commander fleet order start hitch** - assigning GalaxyTrader to a commander with a large Assist fleet processed every subordinate in one MD `do_all`, restarting Assist on each (full MK3 AI cold-start per ship) and causing multi-second freezes. Reactivation now processes **one subordinate per pass**, continues the remainder on `ProcessCommanderGTReactivationYield` with a **10ms** delay (first/settle entry still uses the existing 1s cue), and **skips Assist restart** when the sub already inherits the commander's current GT `baseorder` / `orderobject`. Settle retries for unresolved subs remain bounded (`attempt` lt 5) (`gt_core_system.xml`).
FIX: **Assist "default order not inherited" on load** - vanilla Assist cached `$commander.defaultorder` before its processorder wait; after save-load / AI restart that handle could be dead while `.id` still read `GalaxyTraderMK3`, so inherit skipped and the ship fell back to Follow. Assist now re-reads the live commander default (and Assist baseorder unwrap) after the wait. If inherit still fails while the commander already has a live GT protected order (common with save-load `blocked in run_script but no called script is running`), Assist forces one immediate re-entry at `gt_processorder_immediate` instead of Follow; a one-shot guard prevents loops (`order.assist.xml`).

GalaxyMiner:

CHANGE: **Check Demand First tooltip** - clarifies buy-offer requirement and that storage target only scales priority among offered wares (`{77000,31131}` EN/DE).

Version 0.15.15 (2026-07-17)

GalaxyTrader MK4-2:
FIX: **Shared cache amounts no longer writer-ship sized** - `lib.gt.mk42.tradematch` cached `$Amount` from `offeramount.{refreshing ship}` plus `clamp_trade_amount` with the writer as buyer, so an S-class cache refresh stored ~322-unit rows that every fleet member executed (including XL multitrade). Cache write now uses ship-agnostic bare `offeramount` / `amount` / build-storage deficit and station-side clamp only (seller stock, then destination buyer capacity), matching MK3 `universalCacheBasis`. `trade_found` expands `$Amount` from live `offeramount.{this.ship}` before clamp so stale writer-biased rows fix on read without waiting for cache refresh (`lib.gt.mk42.tradematch.xml`, `order.trade.galaxytradermk42.xml`).
FIX: **Concurrent cache validation no longer shares mutable trade rows** - `lib.gt.mk42.cache.search` appended the cache-resident table from `GT_TradeCacheMK42` into each ship's `GT_Trade_Found` list. Validation and queue then wrote `$Amount` / `$FinalBuyAmount` / `$Queued*` onto that same object, so a Prometheus cargo clamp (e.g. 166 Claytronics) could overwrite a Heron's validated amount (e.g. 2583) before `create_trade_order`. Cache search now hands out a private wrapper copy per ship (offers/stations stay shared game refs); ephemeral clamp fields are omitted and `$Amount` prefers ship-agnostic `$OfferMaxAmount`. Min-cargo util at cache read uses `$OfferMaxAmount` so a peer-poisoned `$Amount` cannot false-reject large ships (`lib.gt.mk42.cache.search.xml`).
